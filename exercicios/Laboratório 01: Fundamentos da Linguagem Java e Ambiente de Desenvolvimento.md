# Laboratório 01: Fundamentos da Linguagem Java e Ambiente de Desenvolvimento																									
																									
## 1. Descrição do Repositório																									
Este repositório destina-se ao desenvolvimento das atividades práticas da aula inaugural da disciplina de Programação Orientada a Objetos (POO). O objetivo central é realizar a transição dos conceitos de lógica de programação para a implementação técnica em Java, utilizando o ambiente de desenvolvimento integrado (IDE) NetBeans.																									
																									
## 2. Objetivos de Aprendizagem																									
* Ambientação com a IDE NetBeans e o fluxo de trabalho Java.																									
* Compreensão da tipagem estrita e declaração de variáveis.																									
* Aplicação de operadores aritméticos e expressões lógicas de comparação.																									
* Prática de entrada e saída de dados via console.																									
* Utilização do GitHub como ferramenta de versionamento e entrega de projetos.																									
																									
## 3. Fontes de Consulta e Apoio																									
Como material complementar para a sintaxe e configuração do ambiente, os alunos devem consultar a playlist oficial:																									
* **Curso de Java para Iniciantes (Prof. Gustavo Guanabara - Curso em Vídeo):** [Acessar Playlist](https://www.youtube.com/playlist?list=PLHz_AreHm4dkI2ZdjTwZA4mPMxWTf1SpM)																									
																									
## 4. Metodologia de Execução e Entrega																									
1. **Fork:** Realize o fork deste repositório para sua conta pessoal no GitHub.																									
2. **Clone:** Clone o repositório para sua estação de trabalho local.																									
3. **Desenvolvimento:** Crie um novo projeto no NetBeans ("Java com Ant") e implemente as classes solicitadas nos desafios abaixo.																									
4. **Submissão:** Realize o commit das alterações e execute o push para o seu repositório remoto. O link do repositório atualizado deve ser enviado ao instrutor conforme as normas da instituição.																									
																									
## 5. Atividades Práticas																									
																									
### Desafio 01: Tipagem e Manipulação de Dados																									
**Arquivo:** `EstudoVariaveis.java`																									
Implemente um programa que declare e inicialize variáveis para os seguintes tipos primitivos e de referência: `String` (nome), `int` (idade), `double` (peso) e `boolean` (status de matrícula). O programa deve exibir esses dados formatados no console através da instrução de saída do sistema.																									
																									
### Desafio 02: Processamento Aritmético																									
**Arquivo:** `CalculadoraOperadores.java`																									
Desenvolva um algoritmo que receba dois valores numéricos. Implemente as cinco operações básicas: adição, subtração, multiplicação, divisão e resto da divisão (módulo). Certifique-se de utilizar tipos de dados que suportem precisão decimal para a operação de divisão.																									
																									
### Desafio 03: Lógica Relacional e Booleana																									
**Arquivo:** `ValidacaoRelacional.java`																									
Implemente um sistema de verificação que utilize uma variável de controle para a maioridade (ex: 18 anos). O programa deve comparar esse valor com uma entrada de usuário e imprimir no console o resultado booleano (true/false) das operações "maior ou igual" e "estritamente igual".																									
																									
## 6. Orientações Técnicas (NetBeans)																									
Para otimizar o desenvolvimento, utilize os snippets e atalhos de teclado da IDE:																									
* **psvm + TAB:** Gera o método `public static void main(String[] args)`.																									
* **sout + TAB:** Gera a instrução `System.out.println()`.																									
* **Alt + Shift + F:** Executa a formatação automática do código de acordo com as convenções da linguagem.																									
* **Ctrl + Espaço:** Aciona o recurso de auto-complete de métodos e variáveis.																									
																									
## 7. Critérios de Avaliação																									
* Integridade da compilação: O código deve executar sem erros de sintaxe.																									
* Nomenclatura: Uso obrigatório da convenção CamelCase (ex: `nomeUsuario`, `calcularMedia`).																									
* Versionamento: Organização das pastas e clareza nas mensagens de commit.
